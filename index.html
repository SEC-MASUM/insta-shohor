JavaScript হচ্ছে client-side scripting language এবং single threader high-level programming language. JavaScript code run করে client side এ। Browser হচ্ছে client-side. Browser-e যখন JS code load করা হয় তখন ব্রাউজার তার JavaScript Engine ব্যবহার করে JS এর heigh-level language কে Machine Language এ convart করে। এর পর সেই Machine কোডকে execute করে output দেয়। বিভিন্ন ব্রাউজার এর JavaScript Engine ভিন্ন ভিন্ন হতে পারে। যেমনঃ Chrome এর JavaScript engines হচ্ছে V8 Engine, Firefox এর Spidermonkey, Edge এর Chakra ইত্যাদি।

Event loop asyncronous way তে কাজ করে। JavaScript Engine এর ভিতর call stack থাকে। JS code এর মধ্যে যখন কোনো function কে call করা হয় তখন সেই function JavaScript Engine এর call stack এর মধ্যে stack আকারে জমতে থাকে। কিন্তু যখন কোনো async function call করা হয় তখন call stack সেই function কে নিজের কাছে না রেখে browser API (Web API) এর কাছে পাঠিয়ে দেয়। browser API সেই funtion এর যা করার সেটা করার পরে event queue (callback queue) তে পাঠিয়ে দেয়। আর বেচারা Event loop সব সময় call stack কে চেক করে দেখে সে ফাকা হয়েছে কিনা। যখনই call stack কে ফাকা দেখে তখনই event queue এর function কে call stack এ পাঠিয়ে দেয় execute করার জন্য। এইটাই চচ্ছে Event loop এর কাজ।

Local storage এর ডাটা browser এ স্থায়ী ভাবে save থাকে, যতক্ষন না সেই ডাটা কে মেনুয়ালি ডিলিট না করা হয়। তাই বাউজার এর ট্যাব বা উইন্ডো ক্লোজ করে দেয়ার পরেও সেই ডাটা থেকে যায়। এই জন্য পরবর্তিতে Local Storage এর ডাটা ব্রাউজার এর যে কোন উইন্ডো বা ট্যাব থেকে থেকে access করা যায়। Local Storage তুলনামূলক Session Storage এর চেয়ে অনেক বেশি মেমোরি(5MB/10MB) ব্যবহার করতে পারে।
Session storage এর ডাটা browser এ স্থায়ী ভাবে save থাকে না। তাই বাউজার এর ট্যাব বা উইন্ডো ক্লোজ করে দেয়ার পরে সেই ডাটা অটোমেটিক ডিলিট হয়ে যায়। এই জন্য পরবর্তিতে Session Storage এর ডাটা ব্রাউজার এর অন্য কোনো উইন্ডো বা ট্যাব থেকে access করা যায় না। Session Storage তুলনামূলক Local Storage এর চেয়ে অনেক কম মেমোরি(4KB) ব্যবহার করতে পারে।
